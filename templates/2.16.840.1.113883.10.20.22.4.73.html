Cognitive Status Problem Observation (DEPRECATED) [2.16.840.1.113883.10.20.22.4.73, release 2014-06-09] view pdf

Description:

A cognitive status problem observation is a clinical statement that describes a patient's cognitive condition, findings, or symptoms. Examples of cognitive problem observations are inability to recall, amnesia, dementia, and aggressive behavior.

A cognitive problem observation is a finding or medical condition. This is different from a cognitive result observation, which is a response to a question that provides insight into the patient's cognitive status, judgement, comprehension ability, or response speed.

THIS TEMPLATE HAS BEEN DEPRECATED AND MAY BE DELETED FROM A FUTURE RELEASE OF THIS IMPLEMENTATION GUIDE. USE OF THIS TEMPLATE IS NOT RECOMMENDED.

Reason for deprecation: Cognitive Status Problem Observation has been merged, without loss of expressivity, into Mental Status Observation (2.16.840.1.113883.10.20.22.4.74).

Conformance Statements:

  • SHALL contain exactly one [1..1] @classCode="OBS" Observation (CodeSystem: HL7ActClass 2.16.840.1.113883.5.6 STATIC) (CONF:1098-14319).
  • SHALL contain exactly one [1..1] @moodCode="EVN" Event (CodeSystem: HL7ActMood 2.16.840.1.113883.5.1001 STATIC) (CONF:1098-14320).
  • MAY contain zero or one [0..1] @negationInd (CONF:1098-14344).
  • SHALL contain exactly one [1..1] templateId (CONF:1098-14346) such that it
    • SHALL contain exactly one [1..1] @root="2.16.840.1.113883.10.20.22.4.73" (CONF:1098-14347).
    • SHALL contain exactly one [1..1] @extension="2014-06-09" (CONF:1098-32600).
  • SHALL contain at least one [1..*] id (CONF:1098-14321).
  • SHALL contain exactly one [1..1] code (CONF:1098-14804).
    • This code SHOULD contain zero or one [0..1] @code="373930000" Cognitive function finding (CodeSystem: SNOMED CT 2.16.840.1.113883.6.96 STATIC) (CONF:1098-14805).
  • SHOULD contain zero or one [0..1] text (CONF:1098-14341).
    • The text, if present, SHOULD contain zero or one [0..1] reference (CONF:1098-15532).
      • The reference, if present, SHOULD contain zero or one [0..1] @value (CONF:1098-15533).

        • SHALL begin with a '#' and SHALL point to its corresponding narrative (using the approach defined in CDA Release 2, section 4.3.5.1) (CONF:1098-15534).

  • SHALL contain exactly one [1..1] statusCode (CONF:1098-14323).
    • This statusCode SHALL contain exactly one [1..1] @code="completed" Completed (CodeSystem: HL7ActStatus 2.16.840.1.113883.5.14 STATIC) (CONF:1098-19091).
  • SHOULD contain zero or one [0..1] effectiveTime (CONF:1098-14324).
    • The effectiveTime, if present, SHALL contain exactly one [1..1] low (CONF:1098-26458).
    • The effectiveTime, if present, MAY contain zero or one [0..1] high (CONF:1098-26459).
  • SHALL contain exactly one [1..1] value with @xsi:type="CD", where the code SHOULD be selected from ValueSet Problem 2.16.840.1.113883.3.88.12.3221.7.4 DYNAMIC (CONF:1098-14349).
  • MAY contain zero or more [0..*] methodCode (CONF:1098-14693).
