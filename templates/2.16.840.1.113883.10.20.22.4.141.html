Handoff Communication Participants [2.16.840.1.113883.10.20.22.4.141] view pdf

Description:

This template represents the sender (author) and receivers (participants) of a handoff communication in a plan of treatment. It does not convey details about the communication. The "handoff" process involves senders, those transmitting the patient's information and releasing the care of that patient to the next clinician, and receivers, those who accept the patient information and care of that patient.

Conformance Statements:

  • SHALL contain exactly one [1..1] @classCode="ACT" Act (CodeSystem: HL7ActClass 2.16.840.1.113883.5.6 STATIC) (CONF:1098-30832).
  • SHALL contain exactly one [1..1] @moodCode="EVN" (CodeSystem: HL7ActMood 2.16.840.1.113883.5.1001) (CONF:1098-30833).
  • SHALL contain exactly one [1..1] templateId (CONF:1098-30834) such that it
    • SHALL contain exactly one [1..1] @root="2.16.840.1.113883.10.20.22.4.141" (CONF:1098-30835).
  • SHALL contain exactly one [1..1] code (CONF:1098-30836).
    • This code SHALL contain exactly one [1..1] @code="432138007" handoff communication (procedure) (CONF:1098-30837).
    • This code SHALL contain exactly one [1..1] @codeSystem="2.16.840.1.113883.6.96" (CodeSystem: SNOMED CT 2.16.840.1.113883.6.96) (CONF:1098-30838).
  • SHALL contain exactly one [1..1] statusCode (CONF:1098-31668).
    • This statusCode SHALL contain exactly one [1..1] @code="completed" Completed (CodeSystem: HL7ActStatus 2.16.840.1.113883.5.14) (CONF:1098-31669).
  • SHALL contain exactly one [1..1] effectiveTime (CONF:1098-31670).
  • SHALL contain at least one [1..*] Author Participation (identifier: 2.16.840.1.113883.10.20.22.4.119) (CONF:1098-31672).
  • SHALL contain at least one [1..*] participant (CONF:1098-31673) such that it
    • SHALL contain exactly one [1..1] @typeCode="IRCP" Information Recipient (CodeSystem: HL7RoleClass 2.16.840.1.113883.5.110) (CONF:1098-31674).
    • SHALL contain exactly one [1..1] participantRole (CONF:1098-31675).
      • This participantRole SHALL contain at least one [1..*] id (CONF:1098-32422).
      • This participantRole SHOULD contain zero or one [0..1] code, which SHOULD be selected from ValueSet Healthcare Provider Taxonomy 2.16.840.1.114222.4.11.1066 DYNAMIC (CONF:1098-31676).
      • This participantRole SHALL contain at least one [1..*] addr (CONF:1098-32392).
      • This participantRole MAY contain zero or one [0..1] playingEntity (CONF:1098-32393).
        • The playingEntity, if present, SHALL contain at least one [1..*] name (CONF:1098-32394).
