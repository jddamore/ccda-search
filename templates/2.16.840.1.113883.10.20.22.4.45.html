Family History Organizer (V3) [2.16.840.1.113883.10.20.22.4.45, release 2015-08-01] view pdf

Description:

The Family History Organizer associates a set of observations with a family member. For example, the Family History Organizer can group a set of observations about the patient's father.

Contained By: Contains:
Health Concern Act (V2)
Risk Concern Act (V2)
Family History Section (V3)
 Family History Observation (V3)

Conformance Statements:

  • SHALL contain exactly one [1..1] @classCode="CLUSTER" Cluster (CodeSystem: HL7ActClass 2.16.840.1.113883.5.6 STATIC) (CONF:1198-8600).
  • SHALL contain exactly one [1..1] @moodCode="EVN" Event (CodeSystem: HL7ActMood 2.16.840.1.113883.5.1001 STATIC) (CONF:1198-8601).
  • SHALL contain exactly one [1..1] templateId (CONF:1198-8604) such that it
    • SHALL contain exactly one [1..1] @root="2.16.840.1.113883.10.20.22.4.45" (CONF:1198-10497).
    • SHALL contain exactly one [1..1] @extension="2015-08-01" (CONF:1198-32606).
  • SHALL contain at least one [1..*] id (CONF:1198-32485).
  • SHALL contain exactly one [1..1] statusCode (CONF:1198-8602).
    • This statusCode SHALL contain exactly one [1..1] @code="completed" Completed (CodeSystem: HL7ActStatus 2.16.840.1.113883.5.14 STATIC) (CONF:1198-19099).
  • SHALL contain exactly one [1..1] subject (CONF:1198-8609).
    • This subject SHALL contain exactly one [1..1] relatedSubject (CONF:1198-15244).
      • This relatedSubject SHALL contain exactly one [1..1] @classCode="PRS" Person (CodeSystem: HL7EntityClass 2.16.840.1.113883.5.41 STATIC) (CONF:1198-15245).
      • This relatedSubject SHALL contain exactly one [1..1] code, which SHOULD be selected from ValueSet Family Member Value 2.16.840.1.113883.1.11.19579 DYNAMIC (CONF:1198-15246).
      • This relatedSubject SHOULD contain zero or one [0..1] subject (CONF:1198-15248).
        • The subject, if present, SHALL contain exactly one [1..1] administrativeGenderCode, which SHALL be selected from ValueSet Administrative Gender (HL7 V3) 2.16.840.1.113883.1.11.1 DYNAMIC (CONF:1198-15974).
        • The subject, if present, SHOULD contain zero or one [0..1] birthTime (CONF:1198-15976).

        • The subject SHOULD contain zero or more [0..*] sdtc:id. The prefix sdtc: SHALL be bound to the namespace �urn:hl7-org:sdtc�. The use of the namespace provides a necessary extension to CDA R2 for the use of the id element (CONF:1198-15249).

        • The subject MAY contain zero or one [0..1] sdtc:deceasedInd. The prefix sdtc: SHALL be bound to the namespace �urn:hl7-org:sdtc�. The use of the namespace provides a necessary extension to CDA R2 for the use of the deceasedInd element (CONF:1198-15981).

        • The subject MAY contain zero or one [0..1] sdtc:deceasedTime. The prefix sdtc: SHALL be bound to the namespace �urn:hl7-org:sdtc�. The use of the namespace provides a necessary extension to CDA R2 for the use of the deceasedTime element (CONF:1198-15982).

        • The age of a relative at the time of a family history observation SHOULD be inferred by comparing RelatedSubject/subject/birthTime with Observation/effectiveTime (CONF:1198-15983).

  • SHALL contain at least one [1..*] component (CONF:1198-32428).
    • Such components SHALL contain exactly one [1..1] Family History Observation (V3) (identifier: 2.16.840.1.113883.10.20.22.4.46) (CONF:1198-32429).
